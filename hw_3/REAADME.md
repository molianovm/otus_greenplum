### Инструкция:

1. Используя датасет из ДЗ в занятии «SQL в Greenlum», напишите следующие запросы:
   1. Query 1: Retrieve Customer Orders with Order and Customer Details
   2. Query 2: Retrieve Detailed Order Information with Line Items
   3. Query 3: Retrieve Supplier and Part Information for Each Supplier-Part Relationship
   4. Query 4: Retrieve Comprehensive Customer Order and Line Item Details
   5. Query 5: Retrieve All Parts Supplied by a Specific Supplier with Supplier Details
2. Замерьте время выполнения, сохраните планы запросов.
3. Используя материалы вебинара, оптимизируйте планы запросов. Примените vacuum, analyze, партиционирования и построение индексов.
4. Сравните время выполнения запросов с индексами и без, соберите информацию по использованию индексов из системных таблиц и представлений.
5. Сделайте вывод о необходимости индексов в Greenplum.

---

### Результат:

## Все задания для каждого из запросов из п.1 (с 1 по 5) выполнены в отдельных файлах (`Query_*.sql`).

### Вывод:

В некоторых случаях использование индексов позволило ускорить запрос при условии использования индексируемого поля в условии фильтрации.

Исходя из опыта работы с Greenplum, для фильтрации могут использоваться достаточно сложные условия, и индекс в таких случаях не будет сильным помощником, а даже наоборот: будет занимать дополнительное место и «занимать время» на вставку.

Более полезным будет правильное распределение таблиц по нужному ключу, поэтапная фильтрация данных во временные витрины с последующем перераспределением по ключу.
